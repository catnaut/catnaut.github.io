    proxy

    powershell 中 可以通过 $env:http_proxy="http://127.0.0.1:10809" 设置代理
    这个效果未知
    reference： powershell 设置代理 | qxdn Blog

    这Windows下代理怎么老是出毛病啊，v2rayN全局也不顶用啊

    无代理情况

    使用v2rayN 在使用 v2fly 核心时的 http 代理

    开启全局后

    在将核心切换回 xray 后问题无法复现，暂时理解为程序bug重启后解决

    此方法对 scoop choco 都无效，据说 vcpkg 可以使用

    powershell

    powershell 设置代理
    netsh winhttp show proxy 查看代理状态

    scoop

    无代理下 aria2 下载出现错误
    scoop关闭 aria2下载 后仍然出现错误

    网上搜索后发现此问题为 网络问题，可以通过代理解决
    reference： scoop 安装出错，相关疑难里中的方法无法解决 · Issue #1273 · neolee/pilot · GitHub

    scoop config proxy "http://127.0.0.1:10809" 在 scoop 内设置 HTTP 代理无效，同样的代理 curl 有效，排除 HTTP代理故障 。查阅文档后发现应该设置为，scoop config proxy "127.0.0.1:10809"

    scoop config rm proxy 删除代理

    突然间，他就可以正常使用了，我完全不知道什么情况啊，莫名奇妙就开始干活了，我绝望了！

    scoop cache 显示已有的软件缓存 scoop cache rm <manifesits> 可以删除指定的缓存

    aria2 下载应该时会自动传递 proxy 配置 可以正常下载，到此网络问题解决
