hexo

介绍

hexo 渲染成静态文件，可以上传到 GitHub pages cloudflare pages 等免费服务商，不但不怕攻击也不用维护。部署也方便，服务器也不会忘记续费资料全没

使用教程

写作

hexo new [layout] <title> 创建新博客
Hexo 有三种默认布局：post、page 和 draft。在创建这三种不同类型的文件时，它们将会被保存到不同的路径；而您自定义的其他布局和 post 相同，都将储存到 source/_posts 文件夹。

draft是一种特殊的布局
hexo publish [layout] <title> 可以将它移动到 source

自定义 模板 scaffold

hexo 会尝试在 scaffold 找模板文件

tags & categories

需要注意的是 在hexo中 分类和tag 不同，分类是有层级的
详情见 Front-matter -分类和标签| Hexo

相对引用图片

开启 post_asset_folder 详情见 配置文件 Writing 部分
使用 ![](<filename>) 引用图片即可，文件放在 与 md 同名在同一目录下文件夹内
